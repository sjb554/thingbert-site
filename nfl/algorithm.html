How the NFL Predictive Algorithm Works

The engine blends matchup-adjusted performance, market baselines, and expected value math to surface actionable plays. Here is the transparent walkthrough.

Step-by-step summary

Step Purpose Example output
1. Data inputs Pull 8-week rolling team stats, closing lines, and injury reports. Yards per play, turnover rate, rest days.
2. Feature engineering Translate raw stats into matchup-adjusted strengths. Offense minus opponent defense differential.
3. Model prediction Predict continuous expected points for each team. Home: 24.3 vs Away: 19.8.
4. Post-processing Convert continuous output into realistic NFL scores. Rounded to 24-20 using score buckets.
5. EV computation Compare model probabilities to the book's implied chances. +3.8% expected edge.
6. Output Publish rated plays with odds, stake size, and predicted score. Displayed in the weekly recommendations table.

Feature engineering breakdown

Sub-step What it does Example
Normalize stats Standardize team metrics to comparable scales. Yards per play z-score = 0.65.
Rolling averages Use eight-week windows to smooth volatility. Raw 24 points becomes 22.7 adjusted.
Matchup differentials Contrast offense versus opponent defensive strength. KC offense 0.8 minus NYJ defense 0.5 = +0.3.
Vegas baselines Decompose spread and total into implied team totals. Spread -3, total 48 -> 25.5 vs 22.5 implied points.
Composite features Blend efficiency, rest, travel, venue, and weather flags. Feature vector X = [net_strength, rest_days, dome_flag].

Expected value math

Component Description Formula / logic
Model probability Win or cover chance from simulation. P_model = 67.2%.
Implied probability Converted from sportsbook odds. P_implied = 1 / (1 + odds/100).
Expected value Edge of the model over the market. EV% = (P_model - P_implied) * 100.
Filters Require minimum edge thresholds. 1% for spreads, 2% for moneyline/totals.

Internal mechanics

  • Logistic regression trained on rolling eight-week samples.
  • Combines team efficiency metrics with Vegas-implied baselines.
  • Produces continuous expected points for mathematical accuracy.
  • Translates points into realistic NFL scoring buckets for presentation.
